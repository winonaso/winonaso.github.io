Design Process

Identify Design Problem

The first step was to determine what area I wanted to redesign - and why. While looking through the site, and speaking to the Interviewers, I realized that Tulip had recently launched the Turnkey solution - and I felt that the fact there were actually three different versions of the application wasn't clear enough from the get go.

I decided that I would flush out the three options so that Tulip clients could better understand the options and flexibility that the Tulip application would give them.

Sketching

I start most of my design work with a basic sketch, even if it's a very rough wireframe of what I want the page to look like. Sometimes the wireframe is done in Photoshop directly, but other times, I want the flexibility of being able to sketch quickly and draw arrows.

Basic HTML

Once I had the sketch done, I had a pretty good idea of how I wanted to layout the page and could begin with basic HTML. The only CSS I added at this point was some background colours so I could ensure that I closed my DIVs correctly.

I started by adding in all the basic text of the page, and the basic frame of the DIVs - I wasn't worried about the page looking pretty quite yet. I also just had placeholder image that lead to dead images as I knew I would need them at some point - this would also make it easier to drop in the images as needed.

You'll also actually notice I didn't close my DIVs properly in the first large image DIV - there is no margin between those. I realize this later one when I start styling the CSS and fix it.

Basic CSS

The next step is to add in the fonts, and removing any of the list decorations, unecessary indentation, and general tidying up of the page. I also began adding margins as needed for the image pages, and adjusting the size of the headers and text.

I also changed the background colours of each DIV to a light grey so I could better see what the page would eventually look like. I added some basic colours - the pink bars, and navigational colours.

Three Solutions

The first peice I started constructing with CSS was the three solutions. I decided to add the more details links directly inside each of the options so it was very clear which option you were getting more details about.

To build this, I used DIVs in order to create each item. Each header was a DIV with the correct background colour, and the entire block was also wrapped in a div to give it the border colour. Additionally, inside the main DIV, I created an unordered list with the different features of the solutions. The more details button was also created by using a DIV, and adding rounded corners via CSS.

Main Content

I cleaned up the main content, adding in images of the iPad (from Tulip's site). I did not choose the main images yet, as I wanted to work on the actual formatting and placing the text properly before I started with the imagery.

Solutions

Next piece to fix up was the solutions. The images were floated left, when they needed to be floated right. By using floats, it also meant I needed to set the overflow to auto in order for the margins on each section to wrap around the items that were being floated.

Another thing I noticed while working on the solutions was that the font wasn't quite right - I had been using AkagiPro-Light instead of AkaiPro - Thin the entire time. This was a quick fix of converting the .otf into a .woff and changing the CSS page to reflect the file name change.

Navigation bar

I added in the look of the main navigation bar as on the main Tulip page with CSS, as well as the social media icons.

Bring it to life/See Tulip in action

On each of the Tulip pages, there are two large buttons at the bottom of each page which I styled and added to my own page. This is where I read up more on using flex in order to make the two large buttons span across the entire width evenly, and center the text in the middle.

Images

I set up the large images on the page to have a DIV on top of the base image with an opacity of 30% in the Tulip dark blue color in order to tint the photo - and had the text on top of the image in another DIV in order to keep the correct colouring of the text. However, I realized that the images on the Tulip site were actually tinted not with CSS, but baked into the images which meant I could remove my tinting DIV, and just leave the images as is.

Footer

This was a simple recreation of the current Tulip footer, and at this point, I was pretty much finished.

Cleaning it Up

This was a matter of cleaning up any weird indentations in my code, editing any typos in the text I may have created, adding in a bit more padding where needed, and just general cleanup.

I also added the little things that really wrap up a page - on hover effects and colour emphasises. I also took a look at existing Tulip pages, as well as the Tulip branding guide to ensure that the correct colours were used, and small touches like the small pink bars were added in.

Tools Used

For this project, I coded the project in Sublime Text 2 (with Solarized - Light color scheme!). The website uses HTML, CSS, and primarily uses DIVs for content organization and positioning. As I worked on the website more, I began to incorporate Flexbox to simplify alignment and adjustment of the different sections of the website.

Colours and icons were edited in Adobe Photoshop. I also used Adobe Photoshop as well as APNG Assembler to create the animated PNG at the top and bottom of this page.

Things I Would Change

If I were to redo this assignment, one major item I would change would be the implementation of the three solutions divs. By using DIVs and floats, I kept on running into weird bugs and errors which were very difficult to fix. If I were to redo this section, I would read up on Flexbox (which I have never used), in order to have each of the solutions acting as a child of a larger parent container instead. This would reduce the headache and difficulties I had using floats and the DIVs in general.

Additionally, I think I would want to brush up on my HTML some more, as using floats in general is not a sustainable or modern way of using DIVs.

Finally, I would also want to implement a mobile-friendly version of my page.
